# 导航与目录

## 左侧导航栏

- 结构来自源目录树：每个子文件夹对应侧栏一组。
- 文件夹的 **首页** 由该目录下的 `index.md`（或 `README.md` 同步而来）充当。
- 本示例中可见四层：**首页 → 指南 / 功能演示 / 参考 → 子页**。
- 侧栏分组标题优先取该组 `index.md` 的第一个 `#` 标题。

试试：折叠 / 展开「功能演示」分组，点击各子页切换（已启用 **instant** 换页：无整页刷新，正文淡入；慢速网络时顶栏有进度条）。

## 右侧本页目录

- 自动根据当前页标题层级生成（`#` / `##` / `###`）。
- 点击可跳到对应小节；长文见 [Markdown 排版](markdown.md)。
- 滚动时当前小节会在目录中高亮（`toc.follow`）。

## 顶栏

| 元素 | 作用 |
| --- | --- |
| 站点名 | 点击回到首页 |
| 搜索框 | 全文检索（见 [搜索](search.md)） |
| 调色板图标 | 浅色 / 深色模式切换 |
| 回到顶部 | 长页滚动后出现 |

## 多层「小仓库」结构示意

```text
example/source/          ← 总源根（本示例）
├── README.md            ← 站点首页
├── guides/              ← 像独立文档包
│   ├── install.md
│   └── advanced/
│       └── config.md
├── showcase/            ← 功能演示包
│   └── …
└── reference/           ← 参考包
    └── glossary.md
```

真实项目中，每个 `guides/`、`reference/` 都可来自不同 Git 仓库，同步到同一源根后一次构建。
